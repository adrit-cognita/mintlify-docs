Mosaic Reporting handles sensitive data including patient health information (PHI), PACS credentials, and authentication secrets. Keeping these values out of source control and Docker images is a foundational security requirement. This page describes where secrets are stored, how they reach running containers, and how local development is set up without exposing production credentials.

<Warning>
  Never log secret values, even at `debug` level. Never commit `.env` files to source control — add `.env` to `.gitignore` and verify it is excluded. Never hardcode secrets in application code, Dockerfiles, or CI workflow YAML. A leaked secret must be rotated immediately; treat any accidental exposure as a security incident.
</Warning>

## Secrets Management Platform

| Environment | Platform | Access Method |
|-------------|---------|---------------|
| `production` | AWS Secrets Manager | ECS task definition secret references by ARN |
| `staging` | AWS Secrets Manager | ECS task definition secret references by ARN |
| `development` | Local `.env` file | Loaded by `dotenv` at process startup |

**AWS Secrets Manager** stores secrets as key-value JSON documents. The ECS task definition references each secret by its ARN and injects it as an environment variable into the container at launch time — the container process never reads from Secrets Manager directly; the ECS agent resolves the reference before the container starts.

If you use Kubernetes, the **ExternalSecrets Operator** syncs secrets from AWS Secrets Manager into native Kubernetes `Secret` objects. Pods reference these Kubernetes Secrets via `envFrom` or volume mounts.

## Secret Inventory

| Secret Name | Description | Used By |
|-------------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string (includes host, port, user, password, database) | Backend API, migration task |
| `REDIS_URL` | Redis connection string (includes auth token) | Backend API |
| `JWT_ACCESS_SECRET` | HMAC-SHA256 signing key for access tokens (min 256-bit entropy) | Backend API (auth module) |
| `JWT_REFRESH_SECRET` | HMAC-SHA256 signing key for refresh tokens (separate from access secret) | Backend API (auth module) |
| `PACS_AE_TITLE` | DICOM Application Entity title for the Mosaic SCU | Backend API (DICOM service) |
| `PACS_HOST` | Hostname or IP of the PACS server | Backend API (DICOM service) |
| `PACS_PORT` | DIMSE port on the PACS server (typically 104 or 11112) | Backend API (DICOM service) |
| `HL7_MLLP_HOST` | Hostname of the HL7 MLLP listener (EMR/RIS) | Backend API (HL7 service) |
| `HL7_MLLP_PORT` | Port of the HL7 MLLP listener | Backend API (HL7 service) |
| `FHIR_SERVER_URL` | Base URL of the FHIR R4 server | Backend API (FHIR service) |
| `FHIR_AUTH_TOKEN` | Bearer token or API key for FHIR server authentication | Backend API (FHIR service) |
| `SMTP_HOST` | SMTP relay hostname for outbound email | Backend API (notification service) |
| `SMTP_USER` | SMTP authentication username | Backend API (notification service) |
| `SMTP_PASSWORD` | SMTP authentication password | Backend API (notification service) |
| `TWILIO_ACCOUNT_SID` | Twilio account identifier for SMS dispatch | Backend API (notification service) |
| `TWILIO_AUTH_TOKEN` | Twilio API authentication token | Backend API (notification service) |
| `SENTRY_DSN` | Sentry Data Source Name for backend error capture | Backend API |
| `AWS_S3_BUCKET_NAME` | S3 bucket name for report PDF and DICOM object storage | Backend API (storage service) |

## Secret Injection in ECS

ECS task definitions reference secrets from Secrets Manager by ARN. At launch time, the ECS agent fetches each secret and injects it as an environment variable — the plaintext value is never written to disk or visible in the task definition JSON.

```json
{
  "containerDefinitions": [
    {
      "name": "mosaic-api",
      "secrets": [
        {
          "name": "DATABASE_URL",
          "valueFrom": "arn:aws:secretsmanager:us-east-1:123456789012:secret:mosaic/prod/database-url-AbCdEf"
        },
        {
          "name": "JWT_ACCESS_SECRET",
          "valueFrom": "arn:aws:secretsmanager:us-east-1:123456789012:secret:mosaic/prod/jwt-access-secret-GhIjKl"
        },
        {
          "name": "JWT_REFRESH_SECRET",
          "valueFrom": "arn:aws:secretsmanager:us-east-1:123456789012:secret:mosaic/prod/jwt-refresh-secret-MnOpQr"
        },
        {
          "name": "REDIS_URL",
          "valueFrom": "arn:aws:secretsmanager:us-east-1:123456789012:secret:mosaic/prod/redis-url-StUvWx"
        }
      ]
    }
  ]
}
```

The ECS task execution role (`ecsTaskExecutionRole`) must have `secretsmanager:GetSecretValue` permission on each referenced secret ARN.

## Secret Injection in Kubernetes

For Kubernetes deployments, the **ExternalSecrets Operator** runs as a cluster-level controller that watches `ExternalSecret` resources and creates corresponding Kubernetes `Secret` objects:

```yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: mosaic-api-secrets
  namespace: production
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: aws-secrets-manager
    kind: ClusterSecretStore
  target:
    name: mosaic-api-secrets
    creationPolicy: Owner
  data:
    - secretKey: DATABASE_URL
      remoteRef:
        key: mosaic/prod/database-url
    - secretKey: JWT_ACCESS_SECRET
      remoteRef:
        key: mosaic/prod/jwt-access-secret
    - secretKey: REDIS_URL
      remoteRef:
        key: mosaic/prod/redis-url
```

Pods then consume the generated `mosaic-api-secrets` Kubernetes Secret via `envFrom`.

## Local Development Setup

In development, secrets are provided through a `.env` file in the repository root. This file is listed in `.gitignore` and must never be committed to source control.

To get started with local development:

1. Copy the example file:

   ```bash
   cp .env.example .env
   ```

2. Fill in your local development values. The `.env.example` file lists every variable with a comment describing its purpose and an example value:

   ```bash
   # PostgreSQL (local Docker)
   DATABASE_URL=postgresql://mosaic:devpassword@localhost:5432/mosaic_dev

   # Redis (local Docker)
   REDIS_URL=redis://localhost:6379

   # JWT (any random string for local dev — do not use production values)
   JWT_ACCESS_SECRET=dev-access-secret-change-me
   JWT_REFRESH_SECRET=dev-refresh-secret-change-me

   # PACS (local DICOM simulator or test PACS)
   PACS_AE_TITLE=MOSAIC_SCU
   PACS_HOST=localhost
   PACS_PORT=11112

   # FHIR (local HAPI FHIR server or test endpoint)
   FHIR_SERVER_URL=http://localhost:8080/fhir
   FHIR_AUTH_TOKEN=

   # Sentry (optional in local dev — leave blank to disable)
   SENTRY_DSN=
   ```

3. Start the dependent services:

   ```bash
   docker compose up -d
   ```

4. Start the backend:

   ```bash
   pnpm dev
   ```

## Secret Rotation

| Secret | Rotation Method | Frequency |
|--------|----------------|-----------|
| `DATABASE_URL` (password portion) | AWS Secrets Manager automatic rotation via Lambda | Every 30 days |
| `JWT_ACCESS_SECRET` | Manual — requires coordinated deployment | Quarterly |
| `JWT_REFRESH_SECRET` | Manual — requires coordinated deployment | Quarterly |
| `TWILIO_AUTH_TOKEN` | Manual — rotate in Twilio console, update in Secrets Manager | Quarterly |
| `SMTP_PASSWORD` | Manual — rotate in email provider, update in Secrets Manager | Quarterly |
| `FHIR_AUTH_TOKEN` | Manual — rotate with FHIR server admin | Per vendor schedule |
| `PACS_*` credentials | Manual — coordinate with PACS administrator | Annually or on staff change |

When rotating a JWT secret, you must deploy the new secret value and restart all running API containers simultaneously. Any access tokens signed with the old secret will immediately become invalid — users will be logged out and must re-authenticate. Schedule JWT secret rotations during a low-traffic maintenance window.
